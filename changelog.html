Recent changes to CIVS

There is no official release of CIVS available. Here is the current snapshot of the source tree. Note that the source tree often contains code that has not yet been pushed to the server. See the README file for restrictions on use.

  • June 28, 2004: Michael Clarkson implemented the new voting and authorization protocol.
  • June 14, 2004: A couple of race conditions were fixed that apparently led to the failure of one high-load (about 2000 voters) election. Also, fixed a bug in reporting results in proportional mode (a set could appear more than once).
  • June 6, 2004: Added tooltips to the result matrices, which makes it easier to see what each number represents. Michael Clarkson also cleaned up the code a fair amount (modernized Perl idioms).
  • May 31, 2004: Added MAM comparison mode under detailed results.
  • May 27, 2004: Michael Clarkson added detailed ballot reporting.
